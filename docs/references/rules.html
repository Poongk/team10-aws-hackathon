참가 규칙

공정하고 즐거운 해커톤을 위한 규칙을 확인해주세요

홈 > 참가 규칙

참가 자격

기본 자격 요건

  • 만 18세 이상의 개발자 (직장인, 프리랜서 모두 가능)
  • 개인 노트북 및 개발 환경 보유
  • 기본적인 프로그래밍 지식 보유
  • 48시간 전 일정 참여 가능

참가 제한

  • 동일인이 여러 팀에 중복 참가 불가
  • 해커톤 진행 중 외부 도움 요청 금지
  • 타 팀과의 코드 공유 및 협업 금지
  • 부적절한 언어나 행동 시 퇴장 조치

팀 구성 규칙

팀 구성

  • 팀 참가: 최소 2명에서 최대 5명까지 구성
  • 사전 팀 구성: 팀 구성을 완료한 후 신청 필수
  • 팀 변경: 개발 시작 후 팀 구성 변경 불가

역할 분담

  • 팀 내 역할 분담은 자유롭게 결정
  • 모든 팀원이 개발에 기여해야 함
  • 발표는 팀 대표 1명이 진행

개발 규칙

필수 사용 도구

  • Amazon Q Developer: 모든 프로젝트에서 필수 사용
  • AWS 서비스: 최소 1개 이상의 AWS 서비스 활용
  • 코드 저장소: 운영측에 공유 가능한 자체 GitHub Repository 준비 필수

개발 제약사항

  • 해커톤 시작 전 작성된 코드 사용 금지
  • 오픈소스 라이브러리 및 프레임워크 사용 가능
  • 외부 API 사용 가능 (단, 유료 서비스는 제한적)
  • AI 도구는 Amazon Q Developer만 사용 가능

제출 요구사항

  • 실행 가능한 프로토타입 또는 MVP
  • 소스 코드 (GitHub 링크)
  • README 파일 (설치 및 실행 방법)
  • 발표 자료 (PPT 또는 데모)
  • Amazon Q Developer 활용 내역 문서화

심사 기준

35점

Amazon Q Developer 활용도 및 창의성

프롬프트 전략과 Amazon Q Developer 와의 상호작용이 효과적이고, 다른 AWS 서비스와의 연계가 잘 이루어졌는지, 새로운 활용의 패턴을 발굴한 사례인지 판단

30점

비즈니스 가치 및 실용성

문제 정의와 타겟 고객층이 명확한지, 수익성과 고객 활용성이 있는지 판단

20점

기술적 구현 완성도

결과물이 안정적으로 일관되게 작동하는지, 제안한 기능의 완성도는 어떠한지, 전체 워크플로우 측면에서 판단

15점

발표 및 데모

실시간 활용 시연과 주요 기능에 대한 시연이 있었는지, 데모의 완성도와 발표 전달력, Q&A 대응력 등을 판단

행동 규범

참가자 의무

  • 다른 참가자에 대한 존중과 배려
  • 공정한 경쟁 정신 유지
  • 행사장 시설 및 장비 보호
  • 주최측 안내사항 준수

금지 행위

  • 다른 팀의 아이디어나 코드 도용
  • 부정한 방법으로 우위 점하기
  • 타인에게 불쾌감을 주는 언행
  • 음주 및 흡연 (지정 구역 외)
  • 상업적 목적의 홍보 활동

지적 재산권

코드 소유권

  • 개발된 모든 코드의 저작권은 참가자에게 귀속
  • 오픈소스 라이선스 적용 권장
  • Amazon은 홍보 목적으로만 프로젝트 소개 가능

데이터 사용

  • 개인정보 및 민감 데이터 사용 금지
  • 공개 데이터셋 사용 권장
  • 데이터 출처 명시 필수

시상 및 상품

상품 지급

  • 상품은 팀단위로 제공
  • 세금은 수상자 개인 부담
  • 상품 지급은 해커톤 종료 후 2달 이내
