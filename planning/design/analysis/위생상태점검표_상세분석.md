# 작업원 일일 위생상태 점검표 상세 분석

## 개요
일동제약 실제 사용 중인 "작업원 일일 위생 상태 점검표"를 기반으로 디지털 체크리스트 시스템 설계

## 실제 양식 분석

### 기본 정보
- **회사**: 일동제약(ILDONG)
- **문서명**: 작업원 일일 위생 상태 점검표 (Check sheet for Personnel hygiene)
- **버전**: Version 3-00
- **작성 주기**: 매일 (출근 전)

### 현재 점검 항목 (9개)

#### 1. 발열, 설사, 구토 등의 증상은 없는가?
**현재 방식**: O/X 체크  
**디지털화**: 
- 유형: 선택형 (있음/없음)
- 위험도: 10 (최고)
- AI 판정: 있음 선택 시 자동 거부

#### 2. 호흡기 질환은 없는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (있음/없음)
- 세부: 기침/콧물/인후통 구분
- 위험도: 9
- AI 판정: 있음 선택 시 재확인 필요

#### 3. 신체 노출부위에 상처는 없는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (있음/없음)
- 추가: 상처 부위 사진 업로드 (있음 선택 시)
- 위험도: 8
- AI 판정: 있음 선택 시 재확인 필요

#### 4. 복장은 깨끗하고 규정에 맞게 착용하였는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (적절/부적절)
- 추가: 작업복 착용 사진 업로드
- 위험도: 5
- AI 판정: 부적절 선택 시 재확인 필요

#### 5. 장신구를 착용하지 않았는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 체크박스 (장신구 착용 안함)
- 위험도: 4
- AI 판정: 미체크 시 재확인 필요

#### 6. 두발/수염머리는 단정한가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (단정함/부적절)
- 위험도: 3
- AI 판정: 부적절 선택 시 재확인 필요

#### 7. 손톱은 짧고한가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (적절/부적절)
- 위험도: 4
- AI 판정: 부적절 선택 시 재확인 필요

#### 8. 진한 화장은 하지 않았는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 선택형 (적절/부적절)
- 위험도: 2
- AI 판정: 부적절 선택 시 경고

#### 9. 작업용 이외의 물품은 반입하지 않았는가?
**현재 방식**: O/X 체크  
**디지털화**:
- 유형: 체크박스 (불필요 물품 반입 안함)
- 위험도: 3
- AI 판정: 미체크 시 재확인 필요

## AI 판정 규칙

### 자동 거부 조건 (출입 불가)
- 증상 = "있음" (발열, 설사, 구토)

### 재확인 필요 조건
- 호흡기 증상 = "있음"
- 상처 = "있음"
- 작업복 = "부적절"
- 장신구 착용 = 미체크
- 두발 = "부적절"
- 손톱 = "부적절"
- 불필요 물품 = 미체크

### 경고 조건
- 화장 = "부적절"

### 의료진 상담 조건
- 증상 + 호흡기 증상 복합
- 상처 + 증상 복합

## 사용자 여정 설계

### 1단계: 체크리스트 시작 (07:30-08:20)
```
작업자가 스마트폰으로 앱 접속
→ "오늘의 위생상태 체크" 버튼 클릭
→ 9개 항목 순차적 입력 시작
```

### 2단계: 항목별 입력 (2-3분)
```
각 항목을 터치 친화적 UI로 입력
→ 필수 항목 미입력 시 다음 단계 진행 불가
→ 실시간 입력 검증
```

### 3단계: 제출 및 AI 분석 (5초 이내)
```
"제출" 버튼 클릭
→ Amazon Q Developer AI 분석 시작
→ 위험도 점수 계산 및 판정 규칙 적용
```

### 4단계: 결과 표시 및 알림
```
판정 결과 화면 표시
→ 작업자에게 푸시 알림 발송
→ 책임자에게 현황 알림 (출입 불가 시)
```

### 5단계: 후속 조치
```
출입 가능: 정상 출근
출입 불가: 귀가 및 휴식 안내
재확인: 관리자 상담 안내
의료상담: 보건실 방문 안내
```

## 부적합자 조치 관리 프로세스

### 조치 필요 판정 기준
**자동 조치 필요 설정**:
- AI 판정 결과가 **자동 거부** 또는 **의료상담**인 경우
- 구체적 조건:
  - 발열/설사/구토 증상 = 있음
  - 복합 증상 (호흡기 + 상처 등)

### 조치 관리 워크플로우

#### 1단계: 자동 감지 및 알림
- AI 판정 완료 즉시 부적합자 조치 필요 상태로 설정
- 운영자에게 실시간 알림 발송
- 해당 작업자에게 "조치 대기" 안내 메시지 표시

#### 2단계: 운영자 조치 작성
**조치 이유 작성**:
- 구체적 판정 근거 (예: "발열/설사/구토 증상 있음")
- 관련 증상 및 위험 요소 명시

**조치 내용 작성**:
- 즉시 조치: "귀가 조치", "의료진 상담", "격리 대기"
- 후속 조치: "해열제 복용", "24시간 휴식", "의료진 진료"
- 복귀 조건: "체온 정상화 후 재검토", "의료진 소견서 제출"

#### 3단계: 책임자 승인
- 조치 내용 검토 및 승인
- 필요시 조치 내용 수정 요청
- 승인 완료 시 조치 확정

#### 4단계: 조치 실행 및 추적
- 해당 작업자에게 조치 내용 안내
- 재검토 일정 자동 설정
- 조치 완료 여부 추적 관리

### 조치 유형별 표준 프로세스

#### 발열/증상 관련 조치
- **조치 이유**: "발열/설사/구토 증상 있음"
- **즉시 조치**: 즉시 귀가, 의료진 상담 권고
- **후속 조치**: 증상 완화까지 휴식
- **복귀 조건**: 증상 완전 소실 후 48시간 경과

#### 복합 증상 조치
- **조치 이유**: "호흡기 증상 + 발열 복합 증상"
- **즉시 조치**: 의료진 상담 필수
- **후속 조치**: 의료진 지시에 따른 치료
- **복귀 조건**: 의료진 소견서 제출 후 검토

## 해커톤 구현 범위

- **간소화된 워크플로우**: 운영자 작성 → 책임자 승인
